Stellarium for Java Migration guide
v 0.1

Jérôme Beau

Abstract:

This document aims to describe the migration of the Stellarium program from C++ to Java.

Introduction

Stellarium is written using C++, which has a lot of common with Java.

The issues are :

  • Typing
    • Booleans
    • Enumerations
  • Templates
  • Function pointers
  • Libraries
    • OpenGL

Instanciations

In C++, failed instanciations return null, which is usually interpreted by Stellarium as a sign for writing an error message and end the program. In Java, instanciations cannot return null. Either they succeed or fail, that is, throw an exception.

Typing

Booleans

Stellarium emulates boolean values though integer values. 0 stands for false, and 1 stands true.

Operator redefinition

Stellarium uses C++ operation redefinition. Java does not support this language feature, but can enable it through operator-like methods.

Enumerations

Stellarium uses C++ enumeration. Java support this language feature since its version 1.5.

LinkedList<s_square> stack;

Parameters by references

Stellarium uses "out" parameters that are modified as multiple returned values. Java does not support this language feature but can emulate it by using references to mutable objects.

Default values

Stellarium uses C++ default values, which allow the developper to avoid providing some parameter values which will be assumed as default values specified it the method prototype. This language feature is not supported in Java, but can be implemented as overloading methods enforcing such default values.

Templates

Stellarium uses the C++ ability to defined parameterized classes, a.k.a templates.

This feature is available in Java since its version 1.5.

Function pointers

Stellarium use the ability of C to use function pointers. Because of its insulation from memory-specific issues, Java doesn't allow such a capability. However, it can be emulated through object design pattern such as Strategies.

GUI

Stellarium implement its own Graphical User Interface. In Java most of these functions are built in the platform, thanks to the AWT and Swing.

Dates

Stellarium implement its own date/time funtions. In Java most of these functions are built in the platform, thanks to the java.util.Date and java.util.Calendar classes.

Librairies

OpenGL

Stellarium uses OpenGL as its 2D and 3D graphic API. In the Java world, OpenGL is available in serveral ways :

  • proprietary APIs
    • JOGL
  • Java3D standard API s

We choosed to use JOGL to ease a first migration step.
